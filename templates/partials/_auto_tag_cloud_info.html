雲端環境偵測 Cloud Environment Detected

目前環境看不到你本機的工具資料夾。掃描功能只能在你自己電腦跑，然後把結果同步到雲端。 Current environment can't access your local project folders. Scanning must run on your own machine, then the result is synced to cloud.
  1. 在本機電腦開終端機（你平常寫 code 的那台）Open terminal on your local machine
    cd path/to/DinoDashboard
python scripts/auto_tag.py --apply
    會掃描每個工具的 path 資料夾，偵測 package.json / requirements.txt / .py / .ts 等，把新技術標籤合併進 tools.yaml Scans each tool's path, detects package.json / requirements.txt / .py / .ts, and merges new tech tags into tools.yaml
  2. 檢查改動Review changes
    git diff tools.yaml
  3. Commit 並部署Commit and deploy
    git add tools.yaml
git commit -m "chore: refresh tech tags"
git push
    push 後你的雲端（Vercel/Railway/Cloud Run 等）會自動重新部署，新標籤立即生效 Your cloud host will auto-redeploy on push; new tags take effect immediately
為什麼不能雲端直接掃描？ Why can't the cloud scan directly?

掃描要讀你每個工具專案的實際檔案，但雲端伺服器沒有你電腦的硬碟。所以「掃描」是開發時動作、「顯示」才是執行時動作 — 你在本機掃一次推上去，雲端永遠看到最新結果。 Scanning reads actual files in each tool's project. Cloud servers don't have access to your local disk. So "scan" is a dev-time action; "display" is the runtime action — scan locally, push, cloud shows the latest.
